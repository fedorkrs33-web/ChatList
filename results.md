## Модель: GigaChat

> Проверить конкретный коммит на GitHub можно несколькими способами:  
>   
> ### 1️⃣ Поиск коммита через интерфейс сайта  
>   
> - Перейди на страницу репозитория: `https://github.com/<имя_репозитория>`.  
> - В поле поиска (обычно вверху страницы) введи идентификатор коммита (`SHA`).  
> - Например, введи: `commit/abcdef1234567890`, где `abcdef1234567890` — SHA-коммит.  
> - Нажми Enter, чтобы перейти на страницу конкретного коммита.  
>   
> ---  
>   
> ### 2️⃣ Проверка коммита через API GitHub  
>   
> Используй REST API GitHub, чтобы получить данные о конкретном коммите:  
>   
> ```bash  
> curl https://api.github.com/repos/<username>/<repository>/commits/<SHA>  
> ```  
>   
> Пример запроса:  
>   
> ```bash  
> curl https://api.github.com/repos/sber/gigachat/commits/abcdef1234567890  
> ```  
>   
> Получишь JSON-ответ с подробной информацией о коммите.  
>   
> ---  
>   
> ### 3️⃣ Использование командной строки Git  
>   
> Можно использовать команду git, чтобы проверить историю изменений и детальную информацию о коммите:  
>   
> ```bash  
> git clone <адрес_репозитория>  
> cd <репозиторий>  
> git log --pretty=format:"%h %an %ad %s" # вывод всех коммитов с SHA, автором, датой и сообщением  
> git show <SHA> # подробная информация о конкретном коммите  
> ```  
>   
0> Например:  
>   
> ```bash  
> git show abcdef1234567890  
> ```  
>   
> ---  
>   
> Таким образом, ты можешь легко найти любой интересующий тебя коммит и узнать подробную информацию о нём.

## Модель: Yandex GPT

> Чтобы проверить коммит на GitHub, выполните следующие шаги:  
>   
> 1. **Перейдите на страницу репозитория:** откройте веб-браузер и перейдите на страницу репозитория на GitHub, где находится интересующий вас коммит.  
>   
> 2. **Перейдите в раздел «commits»:**  
>     * На странице репозитория найдите вкладку «Commits» или «История коммитов» (обычно она находится в меню рядом с вкладкой «Code»).  
>     * Нажмите на эту вкладку, чтобы открыть список всех коммитов для этого репозитория.  
>   
> 3. **Найдите нужный коммит:**  
>     * В списке коммитов вы увидите краткое описание каждого коммита, имя автора и дату.  
>     * Прокрутите список или воспользуйтесь поиском, чтобы найти нужный коммит.  
>   
> 4. **Откройте детальный просмотр коммита:**  
>     * Нажмите на хэш коммита или на краткое описание, чтобы открыть детальный просмотр этого коммита.  
>     * Здесь вы увидите полное сообщение коммита, автора, дату и время коммита, а также список изменённых файлов.  
>   
> 5. **Просмотрите изменения:**  
>     * В детальном просмотре коммита вы можете увидеть, какие именно изменения были внесены в каждый файл.  
>     * Для этого найдите список изменённых файлов и нажмите на файл, чтобы увидеть diff (различия) между версиями.  
>   
> 6. **Используйте дополнительные функции:**  
>     * Вы можете использовать функции сравнения (compare), чтобы увидеть различия между двумя коммитами или ветками.  
>     * Также можно использовать функцию blame, чтобы увидеть, кто и когда вносил изменения в определённый фрагмент кода.  
>   
> Эти шаги помогут вам проверить и проанализировать коммиты на GitHub.
